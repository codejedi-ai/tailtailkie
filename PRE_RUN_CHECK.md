# Pre-Run Checklist ✅

## Final Check Before First Run

### ✅ Files & Scripts
- [x] `build` script exists and is executable
- [x] `stop` script exists and is executable
- [x] `.env` file exists with your secrets
- [x] `env.example` exists (template)
- [x] `docker-compose.yml` exists
- [x] `docker/backend.Dockerfile` exists
- [x] `docker/frontend.Dockerfile` exists

### ✅ Secrets Configuration
- [x] `CLERK_SECRET_KEY` is set in `.env`
- [x] `DATABASE_URL` is set (MongoDB Atlas)
- [x] `MILVUS_URI` is set (Zilliz Cloud)
- [x] `MILVUS_TOKEN` is set
- [x] `MILVUS_USER` is set

### ⚠️ Before Running

1. **Start Docker Desktop** (if not running)
   - Docker must be running before executing `./build`

2. **Verify Docker Compose**
   - Try: `docker compose version` or `docker-compose version`
   - If not found, install Docker Compose

3. **Network Access**
   - Backend needs internet access for:
     - MongoDB Atlas (cloud)
     - Zilliz Cloud (Milvus)
     - Clerk API (for JWT verification)

### 🚀 Ready to Run

```bash
# Build and start everything
./build

# Stop everything
./stop
```

### 📍 Services Will Be Available At

- **Frontend**: http://localhost:3000
- **Backend API**: http://localhost:5000/service/api

### 🔍 Troubleshooting

If `./build` fails:
1. Check Docker is running: `docker ps`
2. Check Docker Compose: `docker compose version`
3. View logs: `docker-compose logs -f`
4. Check `.env` has all required secrets

### 📝 Notes

- Only `frontend` and `backend` services will start (cloud databases)
- Local MongoDB/Milvus are disabled (using profiles)
- `docker-secrets.env` will be auto-generated by `./build`
- Network isolation is relaxed for cloud database access
